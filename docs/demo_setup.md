# 演示数据与角色/项目/批次作用演示指南

本指南用于在答辩或演示前快速铺好数据,使**每个角色、每个单位、每个项目与批次的作用**都能在界面上直观体现。配套脚本:`scripts/seed_demo.sh`。

## 0 前提

在运行 Web 与种子脚本的同一台机器、同一个 shell 中:

```bash
cd /path/to/fileforge
set -a; source /etc/fileforge/fileforge.env; set +a   # 或手动 export DATABASE_URL=...
alembic upgrade head                                   # 确保已建表
```

> 若直接报 `DATABASE_URL 未设置`,说明该 shell 没注入环境变量;Web 的 `/login`(POST)与任何依赖数据库的页面都会因此 500。

## 1 一键铺账号(2 单位 + 4 角色)

```bash
python scripts/seed_demo.py          # 默认口令 Demo@123456789
# 或自定义口令:
DEMO_PASSWORD='你的强口令12位以上' python scripts/seed_demo.py
# 也可用等价的包装脚本:bash scripts/seed_demo.sh
```

脚本创建:

| 账号 | 角色 | 单位 | 能做什么(权限差异) |
| --- | --- | --- | --- |
| padmin | 平台管理员 | 跨单位 | 全部权限;**独有"单位"菜单**,可建单位、跨单位查看 |
| jia_admin | 单位管理员 | 甲 | 管本单位项目/用户;有"用户/整理项目/审计",**无"单位"** |
| jia_op | 单位操作员 | 甲 | 上传跑批 + 修正;**无"用户/整理项目/单位/审计"** |
| yi_op | 单位操作员 | 乙 | 同上,但只能看乙;用于演示跨单位隔离 |

> 脚本幂等:已存在的单位/账号会自动跳过,可安全重复运行;已存在账号的口令不会被改动。

## 2 补项目与批次(在网页里做,顺带演示权限)

1. **jia_admin** 登录 → `整理项目 → 新建` → 建 `项目A1`、`项目A2`(演示 project:manage)。
2. **padmin** 登录 → `整理项目 → 新建` → 替**乙单位**建 `项目B1`(乙只有操作员、建不了项目,正好体现平台管理员的跨单位能力)。
3. **jia_op** 登录 → `上传` → 选 `项目A1` → 上传扫描件 → `开始处理` → 进度页看批次运行 → 批次详情看成功/失败统计。
4. 为演示"**件号按项目独立、跨批次续号**":在 `项目A1` 跑两个批次(件号连续续号),再在 `项目A2` 跑一个(件号重新从 0001 起)。

## 3 推荐的演示输入目录

上传时**一个子文件夹 = 一份多页档案**(压缩包同理,顶层目录=一份档案)。建议准备如下结构,覆盖不同规则路径(放入你自己的真实扫描图):

```
demo_input/
├── 01_普通公文_2023/      # 2020 年后 → 新编码 DQL/ZHL/YWL
│   ├── page_001.jpg
│   └── page_002.jpg
├── 02_简报_春风行动/       # 题名含"简报" → 期限锁定 10 年;若题名文学性 → 触发二次题名重写
│   └── page_001.jpg
├── 03_批评通报/            # 开放状态 → 控制,延期理由"负面信息"
│   └── page_001.jpg
├── 04_工资表/              # 开放状态 → 控制,延期理由"个人隐私"
│   └── page_001.jpg
└── 05_旧件_2008/           # 文件年份 < 2020 → 旧编码 001/002/003
    └── page_001.jpg
```

每一类对应可在演示中点名的"亮点":
- 01:新编码、正常成功路径;
- 02:规则引擎"简报→10 年"期限锁 + 大模型二次题名重写(两阶段);
- 03 / 04:开放状态判定优先级(负面信息 / 个人隐私);
- 05:分类编码随年份在新旧体系切换。

如需让**批次详情出现失败统计**,可额外放一份只含一张极差(空白/严重模糊)图像的文件夹;失败档案会被单独标记、不影响整批,并计入失败分布。

## 4 按角色走查(演示脚本)

1. **平台管理员 padmin**:展示独有的"单位"菜单 → 建单位/账号 → `批次/档案` 里能同时看到甲、乙两边数据(跨单位)。
2. **单位管理员 jia_admin**:有"用户/整理项目"、无"单位" → 建项目、建操作员 → 操作后到"审计记录"看留痕(操作员看不到审计)。
3. **单位操作员 jia_op**:菜单只有 `档案/批次/上传` → 上传跑批 → 批次详情看成功/失败 → 档案检索**只见甲** → 打开档案改题名(archive:correct),修订记录与审计自动留痕。
4. **跨单位隔离**:用 `yi_op` 登录,或把甲某档案的详情 URL 贴到 yi_op 会话 → **返回 404**(单位隔离 + 项目/批次归属校验)。
5. **项目 vs 批次**:用第 2 步第 4 点的件号对比,说明"项目是编号边界、批次是一次运行单元"。
